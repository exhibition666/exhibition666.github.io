Docker安装环境

安装Cent0S7

详情参考另一篇文章：

安装，卸载，启动，停止

查看系统内核

# uname -r

安装

1.切换为root用户

# sudo -i

2.卸载旧版本的docker

# yum remove docker docker-common docker-selinux docker-engine

3.安装所需软件包

# yum install -y yum-utils device-mapper-persistent-data lvm2

4.设置镜像仓库

# yum-config-manager \
--add-repo \
https://download.docker.com/linux/centos/docker-ce.repo

或者设置为阿里云

# yum-config-manager \
--add-repo \
http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo

5.安装最新版Dokcer CE

# yum install docker-ce

6.成功完成以上步骤即为安装成功

卸载

1.卸载软件包

# yum remove docker-ce

2.删除镜像/容器等

# rm -rf /var/lib/docker

启动，停止

启动docker

# systemctl start docker

停止docker

# systemctl stop docker

重启docker

# systemctl restart docker

查看docker状态

# systemctl status docker

开机自动启动docker

# systemctl enable docker

查看版本

# docker version

帮助命令

查看帮助命令

# docker --help

查看概要信息

# docker info

镜像操作

官网镜像搜索 https://hub.docker.com/

列出所有安装镜像

# docker images

所有的镜像都存储在宿主机的 /var/lib/docker 目录下

显示镜像ID

# docker images -q

搜索镜像

# docker search [OPTIONS] 镜像名称

OPTIONS可选参数

-s 列出关注数大于指定值的镜像
# docker search -s 100 centos

--no-trunc 显示完整的镜像描述
# docker search --no-trunc centos

拉取镜像

从Docker Hub拉取镜像

# docker pull 镜像名:标签名

配置国内镜像加速器

阿里云镜像

1.登录阿里云 https://www.aliyun.com/

2.控制台 > 产品与服务 > 容器镜像服务 > 镜像加速器

3.修改daemon配置文件/etc/docker/daemon.json

# vim /etc/docker/daemon.json

4.加入以下内容

{
"registry-mirrors": ["自己阿里云的地址"]
}

5.重启docker服务

# 重载此配置文件
systemctl daemon-reload
# 重启 docker
systemctl restart docker

删除镜像

删除某个镜像

# docker rmi 镜像ID

删除所有镜像

# docker rmi `docker images -q` (注意是反引号)

容器操作

查看

查看正在运行容器

# docker ps

查看所有的容器

# docker ps -a

查看最后一次运行的容器

# docker ps -l

查看停止的容器

# docker ps -f status=exited

创建，启动

1.现有镜像，才能创建容器，拉取镜像

# docker pull centos:7

2.创建容器

docker run [OPTIONS] 镜像名:标签名

[OPTIONS]常用参数

-i ：表示交互式运行容器（就是创建容器后，马上会启动容器，并进入容器），通常与-t同时使用
-t ：启动后会进入其容器命令行, 通常与 -i 同时使用; 加入 -it 两个参数后，容器创建就能登录进去。即分配一个伪终端
--name ：为创建的容器指定一个名称
-d ：创建一个守护式容器在后台运行，并返回容器ID；这样创建容器后不会自动登录容器，如果加 -i 参数，创建后就会运行容器
-v ：表示目录映射, 格式为： -v 宿主机目录:容器目录注意：最好做目录映射，在宿主机上做修改，然后共享到容器上
-p ：表示端口映射，格式为： -p 宿主机端口:容器端口

3.创建交互式容器实例

# docker run -it --name=mycentos centos:7 /bin/bash

退出

# exit

退出但不停止当前容器

Ctrl+p+q

启动、停止

启动单个容器

# docker start 容器名称|容器id

启动所有容器

# docker start `docker ps -a -q` (反单引号)

停止容器

# docker stop 容器名称|容器id

强制停止容器

# docker kill 容器名称|容器id

停止所有容器

# docker stop `docker ps -a -q` (反单引号)

创建守护式进程

# docker run -id --name=名称 镜像名:标签名

例子：
# docker run -id --name=mycentos2 centos:7

登录容器

# docker exec -it 容器名称|容器id /bin/bash

通过exec进入的容器，使用exit只会退出容器，不会停止容器

拷贝宿主机与容器中的文件

# docker cp 要拷贝的宿主机文件或目录 容器名称:容器文件或目录

数据目录挂载

# docker run -id -v /宿主机绝对路径目录:/容器内目录 --name=容器名 镜像名:标签名

挂载只读
# docker run -id -v /宿主机绝对路径目录:/容器内目录:ro --name=容器名 镜像名:标签名

查看容器内部细节

容器所有细节

# docker inspect 容器名称

容器IP地址

# docker inspect --format='{{.NetworkSettings.IPAddress}}' 容器名

删除容器

先停止，再删除

删除单个容器

# docker rm 容器名称 | 容器ID

删除所有容器

# docker rm `docker ps -a -q`

应用部署

MySQL部署

1.拉取镜像

# docker pull mysql:5.7

2.查看镜像是否下载好

# docker images

3.创建容器

-p 代表端口映射，格式为 宿主机映射端口:容器运行端口
-e 代表添加环境变量， MYSQL_ROOT_PASSWORD 是 root 用户的登陆密码 

# docker run -id --name=my_mysql -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 mysql:5.7

4.进入容器，登录MySQL

# docker exec -it my_mysql /bin/bash

#mysql -u root -p

5.远程登录MySQL

如果连接不了，查看防火墙

查看状态：systemctl status firewalld
关闭：systemctl stop firewalld
开机禁用：systemctl disable firewalld

Redis部署

1.拉取镜像

docker pull redis

2.创建容器

docker run -id --name=my_redis -p 6379:6379 redis

3.客户端连接测试

Tomcat部署

1.拉取镜像

docker pull tomcat:8

2.创建容器，并进行目录映射

-p 表示地址映射, 宿主机端口号:容器运行端口号
-v 表示地址映射, 宿主机目录:容器映射目录
–privileged=true 如果映射的是多级目录，防止有可能会出现没有权限的问题，所以加上此参数 

docker run -id --name=my_tomcat -p 8888:8080 -v /usr/local/project:/usr/local/tomcat/webapps --
privileged=true tomcat:8

3.进入Tomcat容器

docker exec -it my_tomcat /bin/bash

4.部署web应用

将web应用放到宿主机相应的目录，即可同步到tomcat容器中

5.测试

RabbitMQ部署

1.拉取镜像

后面带有management就可打开管理界面

docker pull rabbitmq:management

2.创建容器

创建镜像（默认用户名密码）,远程连接端口5672，管理系统访问端口15672，默认用户名： guest ，密码也是 guest 

docker run -id --name=my_rabbitmq -p 5762:5672 -p 15672:15672 rabbitmq:management

启动镜像（设置用户名密码） 

docker run -id --name=my_rabbitmq2 -e RABBITMQ_DEFAULT_USER=username -e
RABBITMQ_DEFAULT_PASS=password -p 5672:5672 -p 15672:15672 rabbitmq:management

3.访问Rabbit管理界面

访问管理界面的地址是 http://[宿主机IP]:15672 , 如：http://192.168.10.11:15672，默认 guest 用户，密码也是 guest
